31  /**
32   * The standardized interface defines an abstraction of the underlaying hardware. The HardwareAbstraction interface defines 
33   * a set of methods that can be implemented by any class anywhere in the class hierarchy. A class that implements 
34   * the interface agrees to implement all the methods defined in the interface, thereby agreeing to certain behavior.
35   * <p>
36   * The functionality provided by the interface is kept as generally as possible in order to not exclude certain
37   * readers. The hardware itself has to comply with the following minimal requirements in order to be supported by the framework.
38   * The minimal assumption about the proprietary RFID systems are:
39   * <ul>
40   * <li>The reader allows the identification of tags by a unique serial number.
41   * </ul>
42   * <p>
43   * The tag's identifiers have to be specified as a String representing a hexadecimal number. The hexadecimal digits are the 
44   * ordinary, base-10 digits '0' through '9' plus the letters 'A' through 'F'. In the hexadecimal system, these digits represent 
45   * the values 0 through 15, respectively. A hexadecimal integer is a sequence of hexadecimal digits, such as 34A7, FF8, 174204. Depending
46   * on the concrete hardware this String will be converted into a convenient format, for example a byte array.
47   * <p>
48   * A tag's memory can be accessed in block units, whereas the blocksize has to be specified in a corresponding properties file.
49   */

94  	 *   <th>
95  	 *    Description
96  	 *   </th>
97  	 *  </tr>
98  	 *  <tr>

100 	 *    n/a
101 	 *   </td>
102 	 *   <td>
103 	 *    continuous
104 	 *   </td>
105 	 *   <td>
106 	 *    When associated with a <code>Source</code>, the trigger fires repeatedly,
107 	 *    as rapidly as possible. When associated with a <code>NotificationChannel</code>
108 	 *    the trigger fires whenever the the report queue transitions from empty to
109 	 *    non-empty.
110 	 *   </td>
111 	 *  </tr>
112 	 *  <tr>
113 	 *   <td>
114 	 *    ms=&ltn&gt
115 	 *   </td>

121 	 *   </td>
122 	 *  </tr>
123 	 *  <tr>
124 	 *   <td>
125 	 *    type=&ltrising|falling&gt ; port=&ltn&gt ; pin=&ltn&gt
126 	 *   </td>
127 	 *   <td>
128 	 *    ioEdge
129 	 *   </td>

132 	 *    The type shall be 'rising' or 'falling'.
133 	 *   </td>
134 	 *  </tr>
135 	 *  <tr>
136 	 *   <td>

156 	 *   <td>

161 	 *   </td>
