29  /**
30   * Signals that a command failed on the hardware abstraction layer. This
31   * exception is thrown by objects within the hardware abstraction layer if the
32   * attempt to execute a command failed. The idea is to propagate just a single
33   * kind of error to the layers above. This simplifies the error handling for
34   * other layers. <n> The error itself can be specified in more detail by using
35   * the following codes: </n>
36   * <ul>
37   * <li><b><code>serviceCode</code>: </b> Indicates the kind of service,
38   * which caused the exception
39   * <li><b><code>readerProtocolErrorCode</code>: </b> The RP conform error
40   * code..
41   * </ul>
42   */
